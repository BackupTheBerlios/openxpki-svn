Download

You can use two different ways to download software from OpenCA. The first option is the file release system from Sourceforge. You have only to go to OpenCA's pages on Sourceforge.net. There you find a link Files where all downloadable files are listed. The second option is OpenCA's own ftp service. Please use one of the mirrors close to you to reduce the load on the lines of ftp.openca.org. This server has only a 2 MBit line today.

Sub Projects

Usually the most people only notice OpenCA as a complete trustcenter system but OpenCA contains three C programs which can be used independently.

All three programs are included in our trustcenter software but all of them have a complete own autoconf/automake system. You can extract them from our releases and use them seperately. They have the same license like OpenCA.

If you need a seperate release of one of these tools then please contact us via the OpenCA users list. We don't publish seperate releases today to reduce the management overhead. This is a volunteer project without a commercial company in the background and so we try to push the most human power into the software and documentation.

OpenCA SV

This small tool can sign, verify, encrypt and decrypt any (binary) data. This include signatures from HTML form made with Internet Explorer or Mozilla. You can use this tool for the encryption of your backups too. The syntax is exactly the same like from OpenSSL. You can find the tool in our sources at src/openca-sv.

OCSPD

The OCSPD implements an OCSP service which can use LDAP as primary source of informations. This allows you to establish an OCSP service for every other PKI platform even if this platform does not support OCSP or requires extra licenses for OCSP daemons. You can find the daemon in our sources at src/ocspd.

SCEP

This tool can be used to create or extract SCEP messages. It uses a syntax like OpenSSL and is used by OpenCA's scripts to provide the SCEP functions operation and getcert. You can find the tool in our sources at src/scep.
